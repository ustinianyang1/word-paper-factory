# 快速入门指南

欢迎使用Word文档自动生成器！本指南将帮助您在几分钟内快速上手并生成您的第一个Word文档。

## 环境准备

### 系统要求

- **Node.js**: 版本 14.0 或更高
- **npm**: 版本 6.0 或更高
- **浏览器**: 推荐使用最新版本的 Chrome、Firefox、Edge 或 Safari

### 检查环境

```bash
# 检查 Node.js 版本
node -v

# 检查 npm 版本
npm -v
```

## 安装步骤

### 1. 克隆或下载项目

如果您有项目源码，可以直接使用。否则，请从GitHub克隆项目：

```bash
git clone https://github.com/your-username/word-paper-factory.git
cd word-paper-factory
```

### 2. 安装依赖

```bash
npm install
```

等待安装完成，可能需要几分钟时间。

### 3. 验证安装

安装完成后，检查是否有错误：

```bash
npm list
```

## 基本操作

### 1. 启动开发服务器

```bash
npm run dev
```

服务器启动后，在浏览器中打开：

```
http://localhost:3000
```

### 2. 创建您的第一个文档

#### 步骤1：填写文档信息

在"文档编辑"标签页中：

1. **输入题目**：在"题目"输入框中输入您的论文题目（必填）
   
   示例：`基于React的Word文档自动生成器研究`

2. **添加个人信息**（可选）：
   
   点击"添加个人信息项"按钮，添加所需的个人信息：
   
   - 标签：`姓名`，内容：`张三`
   - 标签：`学号`，内容：`20240001`
   - 标签：`专业`，内容：`计算机科学与技术`
   
   不同项之间会用单个空格分隔，整体居中显示。

3. **填写摘要**（可选）：
   
   在"摘要"输入框中输入论文摘要，最多500字。
   
   示例：
   ```
   本文研究并实现了一个基于React的Word文档自动生成器。该工具支持自定义格式设置，能够一键生成符合学术规范的Word文档，大大提高了文档编写效率。系统采用现代化的前端技术栈，具有良好的用户体验和可维护性。
   ```

4. **输入关键词**（可选）：
   
   在"关键词"输入框中输入关键词，用分号分隔。
   
   示例：`React；Word文档生成；自动格式；学术规范`

5. **填写引言**（可选）：
   
   在"引言"输入框中输入引言内容。
   
   示例：
   ```
   随着信息技术的快速发展，文档自动化工具在学术和商业领域的重要性日益凸显。传统的Word文档编写方式效率低下，格式不统一，难以维护。因此，开发一个能够自动生成符合学术规范的Word文档的工具具有重要的现实意义。
   ```

6. **填写正文内容**（必填）：
   
   在"正文内容"输入框中输入正文，支持多行输入。
   
   示例：
   ```
   本文设计并实现了一个基于React的Word文档自动生成器。该工具采用现代化的前端技术栈，包括React 18、TypeScript、Vite构建工具和Ant Design UI组件库。系统支持用户自定义各种格式设置，如字体、字号、对齐方式、行距等，能够一键生成符合学术规范的Word文档。
   
   在实现过程中，我们重点解决了以下关键技术问题：
   
   1. 文档生成算法：采用docx库实现Word文档的生成，该库提供了丰富的API，支持复杂的文档结构和格式设置。
   
   2. 状态管理：使用Zustand进行全局状态管理，该库轻量级、性能优秀，API简洁易用。
   
   3. 用户界面：使用Ant Design组件库构建用户界面，该组件库提供了丰富的UI组件，具有良好的用户体验和可访问性。
   
   4. 类型安全：使用TypeScript进行类型检查，确保代码的类型安全，减少运行时错误。
   ```

7. **添加脚注**（可选）：
   
   在"脚注管理"区域：
   
   1. 点击"添加脚注"按钮
   2. 在正文中使用 `[1]`、`[2]` 等格式标记脚注位置
   3. 在脚注输入框中填写脚注内容
   
   示例：
   ```
   随着信息技术的快速发展，文档自动化工具在学术和商业领域的重要性日益凸显。[1]传统的Word文档编写方式效率低下，格式不统一，难以维护。
   ```

8. **填写结论**（可选）：
   
   在"结论"输入框中输入结论内容。
   
   示例：
   ```
   综上所述，本文设计并实现了一个基于React的Word文档自动生成器。该工具具有良好的用户体验和可维护性，能够满足学术文档编写的各种需求。未来的工作可以进一步优化文档生成算法，支持更多的格式选项，提高系统的灵活性和可用性。
   ```

9. **填写参考文献**（可选）：
   
   在"参考文献"输入框中输入参考文献，每条参考文献占一行。
   
   示例：
   ```
   [1] 张三. 基于React的Word文档自动生成器研究[D]. 计算机学报, 2023, 15(3): 123-130.
   [2] 李四. Vite：新一代前端构建工具[J]. 软件学报, 2022, 45(6): 234-239.
   [3] 王五. Ant Design：企业级UI设计语言[J]. 前端开发, 2021, 42(8): 345-352.
   ```

#### 步骤2：自定义格式（可选）

如果您需要自定义格式，切换到"格式设置"标签页：

1. **题目格式**：
   
   - 字体：选择字体（如宋体、黑体、Times New Roman）
   - 字号：选择字号（如三号、小二、二号）
   - 加粗：是否加粗
   - 对齐：选择对齐方式（左对齐、居中、右对齐、两端对齐）

2. **个人信息格式**：
   
   - 字体：选择字体
   - 字号：选择字号
   - 对齐：选择对齐方式

3. **其他部分格式**：
   
   - 摘要标题/内容格式
   - 关键词标题/内容格式
   - 引言标题/内容格式
   - 正文格式（中文/英文）
   - 结论标题/内容格式
   - 参考文献标题/内容格式
   - 脚注格式（中文/英文）

#### 步骤3：查看实时预览

在右侧的"文档预览"面板中，您可以实时查看文档效果。

**注意**：当某个部分（如摘要、关键词、参考文献等）未输入任何内容时，系统会自动隐藏该部分对应的标题，以提升文档预览的专业性和可读性。

#### 步骤4：生成并下载文档

点击右上角的"生成并下载文档"按钮，系统将自动生成Word文档并下载到您的浏览器默认下载目录。

**文件名格式**：`题目.docx`

示例：如果题目是"基于React的Word文档自动生成器研究"，生成的文件名为`基于React的Word文档自动生成器研究.docx`

## 格式设置

### 导出和导入配置

如果您精心设置好了格式，可以保存配置供以后使用：

1. **导出配置**：
   
   - 点击"导出配置"按钮
   - 选择"复制到剪贴板"或"下载文件"
   - 保存配置为JSON文件

2. **导入配置**：
   
   - 点击"导入配置"按钮
   - 选择"粘贴配置"或"从文件导入"
   - 选择之前保存的JSON配置文件

### 重置格式

如果格式设置混乱，可以点击"重置默认"按钮，恢复到默认格式设置。

## 常见问题

### Q1: 安装依赖时出现错误？

**A**: 请尝试以下解决方案：

1. 清除npm缓存：
   ```bash
   npm cache clean --force
   ```

2. 删除node_modules目录并重新安装：
   ```bash
   rm -rf node_modules
   npm install
   ```

3. 检查Node.js版本是否符合要求（14.0或更高）

### Q2: 启动服务器后无法访问？

**A**: 请检查：

1. 防火墙是否阻止了3000端口
2. 是否有其他程序占用了3000端口
3. 尝试使用其他端口或关闭占用端口的程序

### Q3: 生成的Word文档打不开？

**A**: 请尝试：

1. 确保安装了Microsoft Word或兼容的办公软件
2. 检查文件是否完整下载
3. 尝试使用其他浏览器重新下载
4. 检查文件扩展名是否为.docx

### Q4: 预览和生成的文档格式不一致？

**A**: 请检查：

1. 确保在"格式设置"中正确设置了格式
2. 检查预览中是否显示正确
3. 尝试清除浏览器缓存
4. 刷新页面重新生成

### Q5: 如何删除个人信息项？

**A**: 

1. 在"文档编辑"标签页中
2. 找到要删除的个人信息项
3. 点击该项右侧的删除图标（垃圾桶图标）
4. 确认删除

### Q6: 个人信息项之间的分隔符是什么？

**A**: 不同个人信息项之间使用单个空格分隔，整体居中显示。例如：

```
姓名 张三 学号 20240001 专业 计算机科学与技术
```

### Q7: 支持哪些字体和字号？

**A**: 

**字体**：
- 宋体
- 黑体
- Times New Roman
- Arial
- 微软雅黑

**字号**：
- 小五 (9pt)
- 五号 (10.5pt)
- 小四 (12pt)
- 四号 (14pt)
- 小三 (15pt)
- 三号 (16pt)
- 小二 (18pt)
- 二号 (22pt)
- 小一 (24pt)
- 一号 (26pt)

**对齐方式**：
- 左对齐
- 居中
- 右对齐
- 两端对齐

**行距**：
- 1.0倍
- 1.15倍
- 1.5倍
- 2.0倍

### Q8: 如何使用脚注？

**A**: 

1. 在"脚注管理"区域，点击"添加脚注"按钮添加脚注
2. 在正文内容中，使用 `[数字]` 的格式标记脚注位置
   - 例如：`传统的Word文档编写方式效率低下[1]`
3. 在脚注输入框中填写脚注内容
4. 系统会自动在生成的Word文档中创建脚注

### Q9: 摘要和关键词是必填的吗？

**A**: 不是。摘要和关键词都是可选的，可以根据需要填写。如果不填写，生成的文档中不会显示这些部分。

### Q10: 文件名格式可以修改吗？

**A**: 目前文件名格式固定为`题目.docx`，无法自定义。如果需要修改文件名，可以在下载后手动重命名文件。
